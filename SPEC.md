# Техническое задание на Flutter-приложение "Child App"

## 1. Введение и цель проекта

**Child App** — мобильное приложение для родителей, предназначенное для фиксации ключевых моментов развития ребенка от рождения до взросления. Приложение позволяет сохранять текстовые записи, фотографии с тегами, а также отслеживать физические параметры ребенка (рост, вес, размер ноги) на конкретные даты. Все данные хранятся локально на устройстве пользователя, обеспечивая конфиденциальность и автономную работу без необходимости интернет-соединения.

Основная цель приложения — создать единое хранилище воспоминаний и статистики развития ребенка, к которому родители могут обращаться в любой момент для просмотра истории роста и развития. Приложение ориентировано на интуитивно понятный интерфейс, не требующий специальных технических навыков от пользователей.

## 2. Функциональные требования

### 2.1 Текстовые записи (события)

Система должна поддерживать создание текстовых записей с указанием конкретной даты. Каждая запись включает заголовок, текстовое описание и дату события. Пользователь может добавлять произвольные записи типа «сделал первый шаг», «сказал первое слово», «пошел в детский сад» и другие значимые моменты. Записи отображаются в хронологическом порядке в общей ленте событий.

Дополнительно к текстовым записям пользователь может указывать категорию или тип события для удобства фильтрации и поиска в будущем. Категории могут быть предустановленными (первые достижения, здоровье, образование, развлечения) или создаваться пользователем самостоятельно.

### 2.2 Фотогалерея с тегами

Приложение должно обеспечивать возможность добавления фотографий с конкретной датой и тегами. Фотографии можно делать напрямую через камеру устройства или выбирать из галереи. При добавлении фотографии пользователь указывает дату события, изображенного на фото, и произвольные теги для последующего поиска.

Система тегов позволяет маркировать фотографии для быстрого доступа к ним в будущем. Например, теги могут указывать на место (дом, парк, дача), участников (мама, папа, бабушка), или тематику (прогулка, праздник, игра). Теги являются текстовыми и могут создаваться пользователем динамически при добавлении новых фотографий.

### 2.3 Параметры развития (рост, вес, размер ноги)

Приложение должно поддерживать ввод и хранение основных физических параметров ребенка на конкретную дату. Пользователь может записать рост в сантиметрах, вес в килограммах и размер обуви. Эти данные можно вводить как вместе, так и по отдельности — например, только рост или только размер ноги.

Система должна позволять просматривать историю изменения параметров во времени. Рекомендуется реализовать визуализацию в виде простого графика или таблицы, показывающей динамику роста и веса. Это позволит родителям отслеживать физическое развитие ребенка и при необходимости показывать данные врачу.

### 2.4 Лента событий

Главный экран приложения представляет собой ленту всех событий в хронологическом порядке (от новых к старым). Лента объединяет текстовые записи, фотографии и параметры развития в единый поток, позволяя видеть полную картину развития ребенка за любой период.

Каждый элемент ленты отображает тип события, дату и краткую информацию (заголовок записи, миниатюру фото или значение параметра). Пользователь может прокручивать ленту и детально просматривать любой элемент, нажав на него для получения полной информации.

### 2.5 Детальный просмотр событий

При выборе любого события из ленты пользователь переходит на экран детального просмотра. На этом экране отображается полная информация о событии: для текстовой записи — полный текст с заголовком и датой; для фотографии — полноразмерное изображение с датой и всеми тегами; для параметров развития — все введенные значения с датой измерения.

Детальный экран также должен поддерживать редактирование и удаление событий. Пользователь может изменить любую информацию или удалить событие полностью. После редактирования или удаления данные должны корректно обновляться в ленте.

### 2.6 Многопрофильность

Приложение поддерживает несколько детей. При первом запуске создается профиль первого ребенка. Пользователь может добавлять произвольное количество детей. Каждый профиль содержит: имя (обязательно), дата рождения (обязательно), пол (обязательно, male/female), фото (опционально). Переключение между детьми осуществляется через DropdownButton в AppBar.

## 3. Нефункциональные требования

### 3.1 Локальное хранение данных

Все данные хранятся исключительно локально на устройстве пользователя. Для реализации используется SQLite (пакет sqflite). Схема базы данных включает таблицы для детей, событий, фотографий, тегов, параметров развития и настроек напоминаний.

### 3.2 Производительность

Приложение должно запускаться за время не более 3 секунд на устройствах среднего класса. Загрузка ленты событий происходит плавно, с постепенной подгрузкой данных при прокрутке. Работа с фотографиями использует thumbnails для отображения в ленте и полноразмерные изображения на экране детального просмотра.

### 3.3 Интерфейс и UX

Используется Material Design 3 с адаптивной цветовой схемой. Приложение поддерживает темную и светлую темы. Навигация осуществляется через нижнюю навигационную панель с основными разделами: лента, фотографии, параметры, настройки. Все экраны интуитивно понятны и не требуют инструкций для использования.

### 3.4 Напоминания

Настраиваемые уведомления о необходимости замерить параметры ребенка. Пользователь настраивает периодичность (еженедельно, раз в 2 недели, ежемесячно) и время. Реализация через `flutter_local_notifications`.

### 3.5 Совместимость с Android 14+

Целевая версия — Android 14 (API 34). Учет новых требований к разрешениям (granular media permissions, foreground service type declarations).

## 4. Архитектура приложения

### 4.1 Структура проекта

Используется чистая архитектура (Clean Architecture) с разделением на слои: presentation (UI и виджеты), domain (бизнес-логика и модели), data (репозитории и источники данных). Навигация через go_router.

Для управления состоянием используется flutter_bloc.

### 4.2 Основные экраны

| Экран | Описание |
|-------|----------|
| HomeScreen | Главный экран с хронологическим списком всех событий |
| DetailScreen | Полная информация о выбранном событии с возможностью редактирования |
| EventEditScreen | Форма для ввода текстовых событий |
| PhotoEditScreen | Выбор фото, ввод даты и тегов |
| GrowthScreen | Ввод и просмотр роста, веса, размера ноги с графиком динамики |
| SettingsScreen | Выбор темы, управление данными, настройки напоминаний |
| ChildrenListScreen | Управление профилями детей |
